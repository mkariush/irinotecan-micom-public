# Gut-microbial SN-38 reactivation capacity across colorectal-cancer cohorts

Community metabolic modelling (MICOM on AGORA2 v2.01) of gut-bacterial **β-glucuronidase
SN-38 reactivation** — the deconjugation of the inactive irinotecan metabolite SN-38-glucuronide
back to toxic SN-38 — across **nine publicly available colorectal-cancer (CRC) shotgun-metagenomic
cohorts (1,509 primary samples)** accessed through `curatedMetagenomicData`.

This repository contains the full analysis pipeline (R + Python) behind the manuscript.

## Key findings

- Predicted reactivation **capacity reduces almost entirely to a single quantity** — the summed
  abundance of β-glucuronidase-carrying taxa (capacity ≈ 100 × carrier abundance, r² ≈ 0.99) —
  and is unchanged by community growth or oxygen constraints.
- Capacity is carried by recurring high-prevalence commensals and is **not a CRC biomarker**:
  individual cohorts give significant differences in *opposing* directions that cancel under
  meta-analysis.
- Placed on an absolute scale, **microbial capacity exceeds host-delivered substrate by 3–5 orders
  of magnitude**, so realised intestinal reactivation is **host-gated**: substrate delivery, not
  microbial composition, sets the rate for all but a carrier-deficient minority.

## Repository layout

```
R/        cohort acquisition + taxonomy preparation (curatedMetagenomicData)
python/   AGORA2 prep, MICOM model construction, capacity/flux computation,
          statistics, figures (make_fig1..make_fig6) and supplementary tables
data/     placeholder tree (.gitkeep); raw inputs and large artefacts are NOT
          redistributed here — see "Data availability" below
environment.yml          conda environment for the Python pipeline
renv.lock / .Rprofile / renv/   pinned R dependencies (renv)
```

## Requirements

- **Python** environment from `environment.yml` (MICOM, COBRApy, pandas, numpy, matplotlib,
  seaborn). Create with `conda env create -f environment.yml`.
- **R** with `renv`; run `renv::restore()` to reproduce the pinned library, including
  `curatedMetagenomicData`.
- **AGORA2 v2.01** strain reconstructions, obtained from the
  [Virtual Metabolic Human / AGORA2 resource](https://www.vmh.life/). The scripts expect the
  reconstruction directory at a local `databases/AGORA2_json` (and `AGORA2_SBML` / `AGORA2_mat`);
  set this path at the top of the relevant scripts to your local copy.

| Stage | Scripts |
|-------|---------|
| 1. Cohorts & taxonomy | `R/01_fetch_crc_data.R`, `R/02_prepare_taxonomy.R` |

| 4. Analysis | `06_statistics.py`, `10_results.py`, `13_r6_refined.py`, `14_forest.py`, `16b_substrate_sweep.py`, `22_yachida_heinken.py`, `23_depth_sensitivity.py`, `24_absolute_units.py`, `25_phase_diagram.py` |
| 5. Figures | `make_fig1.py` … `make_fig6.py` (shared GUS-class weighting in `gus_efficiency.py`) |

The `R/99*` scripts are auxiliary cohort-overlap checks; `find_sn38_reactions.py` (provenance for the
AGORA2 SN-38-glucuronide reaction IDs), `11_stranded_gus.py` (a reconstruction-coverage validation),
and `17b_q1_plot.py` (an alternate combined version of the Fig 5 panels) sit outside the main
pipeline. Each `make_figN.py` has a `STYLE` block at the top and writes vector (SVG) panels for
external composition.

## Data availability

- **Metagenomes**: the nine CRC shotgun-metagenomic cohorts are obtained programmatically through the
  [`curatedMetagenomicData`](https://bioconductor.org/packages/curatedMetagenomicData/) Bioconductor
  package (Pasolli et al., *Nat Methods* 2017); the exact cohort set is defined in
  `R/01_fetch_crc_data.R`. The cohorts derive from the original studies of Zeller et al. 2014,
  Yu et al. 2017, Feng et al. 2015, Thomas et al. 2019, Wirbel et al. 2019, Vogtmann et al. 2016 and
  Yachida et al. 2019 (primary), and Gupta et al. 2019 and Hannigan et al. 2018 (read-depth
  sensitivity). No patient-level sequence data is redistributed here.
- **Metabolic reconstructions**: AGORA2 v2.01, from the Virtual Metabolic Human resource
  (https://www.vmh.life/). These are not redistributed here.
- Intermediate artefacts (built community models, flux tables, parquet caches) are git-ignored and
  regenerated by the pipeline.
